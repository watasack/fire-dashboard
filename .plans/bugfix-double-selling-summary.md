# FIRE後の現金蓄積バグ修正サマリー

## 問題の概要

**症状**: FIRE後から現金が毎月27万円ずつ増加し、2042年9月の段階で株式がゼロになる

**根本原因**: FIRE後の収入（年金+副収入）があるにも関わらず、無条件に株式を売却していたため、現金が過剰に蓄積

## 根本原因の詳細

### 修正前の処理フロー（バグあり）

```
1. 収入を現金に加算（年金+副収入: ~20万円）
2. FIRE後の現金管理（_manage_post_fire_cash）
   → 月初に無条件に生活費1ヶ月分（27万円）を株式から売却
3. 支出を現金から引き出し（27万円）
4. 運用リターンを適用
```

### 問題の具体例

**初期状態**: 現金500万円（安全マージン）、株式6300万円

**1ヶ月目の処理（修正前）:**
1. 収入: 現金500万 + 20万 = 520万
2. **現金管理: 株式から27万円売却（無条件）** → 現金547万
3. 支出処理: 現金547万 - 27万 = 520万
4. 運用リターン: 株式に適用
5. **結果**: 現金520万（+20万増加）、株式6272万

**2ヶ月目の処理（修正前）:**
1. 収入: 現金520万 + 20万 = 540万
2. **現金管理: 株式から27万円売却（無条件）** → 現金567万
3. 支出処理: 現金567万 - 27万 = 540万
4. 運用リターン: 株式に適用
5. **結果**: 現金540万（+20万増加）、株式6245万

→ **毎月、収入があるにも関わらず株式を売却** → 現金が毎月20万円ずつ蓄積
→ **実際には収入だけで支出の大部分をカバーできている**

いや、待って。上記の分析は間違っている。

もう一度考え直すと...

**1ヶ月目の処理（修正前）:**
1. 支出処理: 現金525万 - 25万 = 500万（株式売却なし）
2. 現金管理: 株式から25万円売却 → 現金525万
3. 結果: 現金525万、株式975万（25万減少）

**2ヶ月目の処理（修正前）:**
1. 支出処理: 現金525万 - 25万 = 500万（株式売却なし）
2. 現金管理: 株式から25万円売却 → 現金525万
3. 結果: 現金525万、株式950万（さらに25万減少）

一見問題ないように見えるが、実は**現金管理の目的が達成されていない**。

現金管理の目的は「安全マージン500万 + 生活費1ヶ月分25万 = 525万を維持する」こと。
しかし、支出処理後に現金管理を実行すると、毎月25万円を売却しているのに、
それは「すでに使ってしまった25万円を補充する」ためだけに使われている。

つまり、**安全マージンを維持するための仕組みが、単なる支出補填になってしまっていた**。

正しくは:
1. **まず現金管理で生活費を準備** → 現金バッファを確保
2. **そのバッファから支出を払う** → 株式売却不要

## 修正内容

### 修正後の処理フロー

```
1. 収入を現金に加算（年金+副収入: ~20万円）
2. FIRE後の現金管理（_manage_post_fire_cash）
   → 現金が目標レベル（525万円）を下回っている場合のみ株式売却
3. 支出を現金から引き出し（27万円）
4. 運用リターンを適用
```

### 修正の核心ロジック

**変更前（バグ）:**
```python
# 月初に無条件に生活費1ヶ月分を売却
if is_start_of_month:
    monthly_cash_need = monthly_expense
    # 株式を売却...
```

**変更後（修正）:**
```python
# 目標現金レベル: 安全マージン + 生活費1ヶ月分
target_cash_level = safety_margin + (monthly_buffer_months * monthly_expense)

# 現金が目標レベルを下回っている場合のみ株式売却
if is_start_of_month:
    cash_shortage = target_cash_level - cash
    if should_sell_monthly and cash_shortage > 0 and stocks > 0:
        # 不足分だけ株式を売却...
```

### 修正箇所

**simulator.py: _manage_post_fire_cash()** (行1770-1794付近)
- 無条件売却 → 目標レベルチェック後に不足分のみ売却

### 修正後の動作（正しい動作）

**1ヶ月目の処理（修正後）:**
1. 収入: 現金500万 + 20万 = 520万
2. 現金管理: 目標525万 - 520万 = 5万不足 → **株式から5万円のみ売却** → 現金525万
3. 支出処理: 現金525万 - 27万 = 498万
4. 運用リターン: 株式に適用
5. **結果**: 現金498万（収支バランスが適切）

**2ヶ月目の処理（修正後）:**
1. 収入: 現金498万 + 20万 = 518万
2. 現金管理: 目標525万 - 518万 = 7万不足 → **株式から7万円のみ売却** → 現金525万
3. 支出処理: 現金525万 - 27万 = 498万
4. 運用リターン: 株式に適用
5. **結果**: 現金498万（安定維持）

→ **収入と支出のバランスを考慮し、不足分のみ株式売却**

## まとめ

**修正前**: 収入の有無に関わらず無条件に株式を売却 → 現金が過剰に蓄積、株式が過度に減少

**修正後**: 収入を考慮し、現金が目標レベルを下回っている場合のみ不足分を株式売却

**修正の効果**:
- 現金が安全マージン（500万円）で安定維持
- 株式売却が最適化され、資産が適切に保全される
- 年金・副収入がある場合も正しく機能する

この修正により、FIRE後の現金管理戦略が収入のある現実的なケースでも正しく動作するようになった。
