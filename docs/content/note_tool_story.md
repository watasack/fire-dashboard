# 共働きFIREシミュレーターを作った話——育休前の「計算できない不安」から始まった

#FIRE #共働きFIRE #育休 #子育て #シミュレーター

---

妻の育休が近づいてきたとき、FIREの計算が一度できなくなりました。

それまでは「年収×貯蓄率×年数」みたいな単純計算でなんとなく見通せていたんですが、
子どもが生まれると変数が一気に増えるんですよ。

- 妻の育休中の収入はいくらになるか
- 育休後の時短勤務でどれだけ収入が下がるか
- 保育料や教育費がいつからいくらかかるか
- 育休を取る前と後で、FIREの時期がどれだけずれるか

これを全部まとめて計算できるツールが、見当たりませんでした。

---

## 「自分で作るか」

エンジニアのよくない癖で、なければ作り始めます。

最初はPythonスクリプトで、育休中の収入変化を月次で追うだけのものでした。
それが気づいたら、こうなっていました。

- 産前・産後・育休・時短勤務の期間を個別に設定できる
- 夫・妻それぞれの雇用形態（会社員／個人事業主）に応じて年金・昇給を自動計算
- 子どもの誕生日と教育方針（公立／私立）を入力するだけで教育費を自動積算
- 住宅ローン（返済額・完済年）か賃貸（月額家賃）を選択できる
- 1,000通りの市場シナリオでFIRE成功確率を算出するモンテカルロシミュレーション

本業が数理モデリングなので、「確率で考える」のが自然な発想でした。

「FIREできるか」ではなく、「**何%のシナリオでFIREできるか**」を計算する。
リーマンショック級の暴落が来ても資産が持つかどうかを、1,000通りで検証する。

こういう作り方になったのは、仕事の延長線上にある感覚です。

---

## 実際に自分の行動が変わった

このシミュレーターを使って、妻と一緒にいくつかの意思決定をしました。

**育休の取得期間**
「夫も育休を12ヶ月取ると、FIREが何年遅れるか」を計算しました。
結果は約2年。「2年の遅れ」と「子どもとの時間」を数値で比較したことで、
感情論ではなく、納得感のある判断ができました。

**支出の見直し**
感度分析（「支出を月3万円減らすとどうなるか」）を回したら、
収入を増やすより支出を減らすほうが、FIREへの影響が大きかった。
この結果を見て、固定費の見直しを本格的にやりました。

**NISA積立額の変更**
育休中の収入減を考慮した上で、「NISA積立額をいくらにすれば無理なく続けられるか」を
シミュレーションして決めました。

「自分と同じ状況の人に使ってもらえたら、同じように役立つんじゃないか」と思ったからです。

共働き・子育て世帯で、FIREを本気で考えている人。
育休の影響が気になっている人。
「自分でExcelで計算しようとしたけど、変数が多すぎて挫折した」人。

そういう人向けのツールです。
