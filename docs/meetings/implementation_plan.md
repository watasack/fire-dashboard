## フェーズ1：バグ修正（最優先・即座に対応）

### BUG-01: フリーランス年収上昇率デフォルト
**場所**: `components/ConfigPanel/IncomeTab.tsx` または `lib/simulator.ts`
- 雇用形態が「自営業」「フリーランス」の場合、`incomeGrowthRate` の初期値を `0%` に設定
- 雇用形態変更時のハンドラーで年収上昇率を連動更新

### BUG-02: 専業主婦の退職年齢バグ
**場所**: `lib/simulator.ts`（計算ロジック）
- 雇用形態が「専業主婦/主夫」の場合、退職年齢を計算に使用しないよう条件分岐を追加
- 現状は退職年齢がシミュレーションに影響してしまっている

### BUG-03/04: 右パネルの連動不整合・FIRE成功確率の差異
**場所**: 右パネルのコンポーネント
- 右パネルの世帯年収・FIRE成功確率等が入力値の変化に追随していない箇所を調査
- 根本原因: 計算結果のどの値を参照しているか確認し、KPIパネルと同一ソースに統一

### BUG-05: SWR値のリアルタイム非更新
**場所**: KPIパネルの必要資産額説明文
- 説明文にSWR値をハードコードしている場合は動的参照に変更
- または必要資産額説明内のSWR数値表記を削除（推奨）

### BUG-06: マイナス値の万円単位消失
**場所**: 収支グラフのフォーマッター
- `Intl.NumberFormat` 等の数値フォーマット関数でマイナス値の場合でも単位を付与するよう修正

## フェーズ3：機能削除・構造変更（中規模）

### 3-1: シミュレーション期間スライダーの削除
- 詳細タブから`simulationPeriod`スライダーを削除
- `lib/simulator.ts` で100歳まで固定計算するよう変更
- テスト：シミュレーション結果が変わらないことを確認

### 3-2: 右パネルの整理
- 「世帯年収・貯蓄率・子育て費用」を右パネルから削除
- FIRE成功確率の重複表示を削除（KPIパネルに集約済みのため）
- 目標達成率は現状維持

### 3-3: MCリターンモデル選択肢の集約
- 正規分布・平均回帰・ブートストラップの3択を廃止
- デフォルト（正規分布等）に固定、UIから選択肢を削除
- `lib/simulator.ts` のMCモデル処理は維持、パラメータをハードコード

### 3-4: 産休育休入力欄をライフタブに移動
- `IncomeTab.tsx` から産休育休関連コンポーネントを削除
- `LifeTab.tsx` に移動（子供数スライダーの近くに配置）
- 状態管理（Context/Zustand等）の整理

### 3-5: SWR廃止（要最終確認）
- 投資タブのSWRスライダーを廃止
- KPIパネルの必要資産額計算では4%固定（またはデフォルト4%で詳細設定可）
- 廃止する場合はシミュレーターエンジンのSWR参照箇所を固定値に変更

- 移動後もシミュレーターへの入力は変わらないため機能影響なし

## フェーズ4：機能追加（大規模）

### 4-1: 現金・株式・NISA入力箇所の統一
**要決定**: 基本タブか投資タブか
- 推奨: **投資タブに統一**（NISAスイッチが投資タブにあるため）
- 基本タブの「詳細入力」で現金/株式を分けて入力する現機能を投資タブに移動
- NISA残高は投資タブで株式入力と合わせて入力

### 4-2: 住居設定の大幅拡充
新規コンポーネント `HousingSection.tsx`（ライフタブ内）
```
持ち家/賃貸 選択
├── 賃貸: 月額家賃 [入力]
│         将来の持ち家取得予定 [有/無]
│         └── 取得予定年: [入力]、物件価格: [入力]
└── 持ち家: 月額ローン返済 [入力]
           詳細入力 ▼
           ├── 固定/変動金利 選択
           ├── 残高・残年数 [入力]
           ├── 固定資産税（年額）[入力]
           └── 大規模修繕積立（年額）[入力]
```
- `lib/simulator.ts` に賃貸費用・住宅取得タイミング・修繕費の計算ロジック追加

### 4-3: 教育費の個別設定化
現状: 公立・オール私立・私立の3択
変更後:
```
子供ごとに設定:
├── 保育園/幼稚園: 認可公立 / 認可私立 / 認可外 [選択] + 月額[任意入力]
├── 0〜2歳保育料: [任意入力]
├── 小学校: 公立 / 私立 [選択]
├── 中学校: 公立 / 私立 [選択]
├── 高校:   公立 / 私立 [選択]
└── 大学:   公立 / 私立 / 医大 [選択]
```
- 各段階にデフォルト値を設定し、個別修正可能な入力欄を追加
- `lib/simulator.ts` の教育費計算ロジックを段階別計算に変更

### 4-4: 児童手当の現行制度対応
- 現行の児童手当制度（2024年10月改正後）を調査
- 所得制限撤廃・支給額・支給期間の最新値を反映
- 所得に基づく自動計算ロジックをシミュレーターに実装

### 4-5: NISA制度の最新化
- 2024年以降の新NISA（成長投資枠・積立投資枠）を調査
- 旧積立NISA廃止に対応
- 投資タブの表示・計算ロジックを更新

### 4-6: 誕生月の入力・計算への反映
- 各人物の誕生月入力欄を追加
- シミュレーターで年単位ではなく月単位でのFIRE達成判定に対応

## フェーズ5：スマホUI大改修

### 現状の問題
- アコーディオン形式で入力欄が縦に並んでいる
- グラフとKPIを見るために大きくスクロールが必要

### 新デザイン
```
スマホ画面（常時表示）:
┌─────────────────────┐
│  KPIバナー          │ ← 常時固定
│  FIRE達成: ○○歳    │
├─────────────────────┤
│  資産推移グラフ     │ ← メインビュー
│  ／収支タブ        │
│  ／年次表タブ      │
│  ／次の一手タブ    │
├─────────────────────┤
│ [基本][収入][投資][ライフ][詳細] │ ← 下部ナビ
└─────────────────────┘

タブをタップすると:
┌─────────────────────┐
│  KPI（縮小表示）    │
├─────────────────────┤
│                     │
│  入力欄             │ ← ボトムシート
│  （上にスワイプで   │    が下から出現
│   フルスクリーン）  │
│                     │
└─────────────────────┘
```

**実装方針**:
- `shadcn/ui` の `Sheet` コンポーネント（または `Drawer`）をボトムシートとして使用
- `lg:hidden` のモバイルレイアウト専用コンポーネントを新規作成
- 既存のデスクトップレイアウトとは完全分離
