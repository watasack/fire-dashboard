## 残タスク一覧

### A. 技術・インフラ（Week 1 目標）

- [ ] **A-1. デモ版↔フル版の統合テスト**
  - ブラウザでデモ版の制限（タブロック・グラフぼかし）を確認
  - アクセスコード入力 → フル版解放の一連の流れを確認
  - ネットワーク障害時のフェールオープン挙動を確認

- [ ] **A-4. 独自ドメインの取得・設定**
  - `.com` ドメインの取得（年1,000〜2,000円）
  - Vercel のカスタムドメイン設定・DNS設定
  - HTTPS の確認

### B. コンテンツ準備（Week 1〜2 目標）

- [ ] **B-1. スクリーンショット撮影**
  - ケース別3パターン（育休なし / 妻のみ育休 / 夫婦育休）
  - ツール全体（`result_top.png` / `full_page.png` / `mobile_top.png` / `mobile_chart.png`）
  - 記事内の `<!-- 📷 -->` コメント位置に対応

- [ ] **B-2. サムネイル画像作成**
  - note記事4本分のサムネイル
  - 方法: Canva or ツールスクショ加工

- [ ] **B-3. note記事のプレースホルダー埋め込み**
  - 全4記事の `[デモURL]` → 本番URL
  - `note_article_draft.md` の `[note有料記事URL]` → 販売記事のnote URL
  - `note_sales_article.md` の `[育休×FIRE記事URL]` → article_draftのnote URL
  - `note_sales_article.md` の `○○○円` → 販売価格（¥980想定）
  - `note_tool_story.md` の `2026年○月` → ツール公開時期

- [ ] **B-4. note有料記事の設定・公開**
  - 価格設定: ローンチ価格 ¥980（5件売れたら ¥1,980 に値上げ）
  - 有料エリアの境界線設定
  - 公開順（同日に投稿）:
    1. `note_self_intro.md` — 自己紹介・信頼構築
    2. `note_tool_story.md` — 開発ストーリー・共感
    3. `note_article_draft.md` — 育休×FIRE計算・集客
    4. `note_sales_article.md` — 有料販売

- [ ] **B-5. X 初回投稿**
  - デモURL + note記事リンク付き
  - `docs/content/x_posts_drafts.md` の下書きを使用

### C. 補助導線（Week 2 以降・後回し可）
