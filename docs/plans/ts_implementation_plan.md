### Phase 4: 資産管理の高度化

> P1後、4A〜4D は並行実施可能。4D は 4A が前提。

#### 4A: 現金・株式分離管理

- `SimulationConfig.currentAssets` → `cashAssets + stockAssets` に分割
- `stocksCostBasis`（課税口座の取得原価）追加
- 毎年の余剰現金を自動投資（NISA優先→課税口座）
- 不足時は株式から取り崩し（NISA優先→課税口座）

#### 4B: NISA年間枠管理 ＋ 売却優先

- 年間投資額が360万円枠を超えないよう制御
- 不足資金発生時: NISA口座から優先売却（非課税）→ 超過分は課税口座から補填
- `nisaCostBasis`（NISA口座の取得原価）追加

#### 4C: iDeCo 60歳制約

- `idecoAssets` は `person1Age < 60` の間は取り崩し不可フラグを追加
- 60歳以降に解放、受取時課税計算（退職所得控除等）は簡略化して実装

#### 4D: 株式売却税（20.315%）

- 課税口座からの取り崩し時に含み益割合で課税:
  ```
  譲渡益 = 売却額 × (1 − 取得原価/時価)
  税額   = 譲渡益 × 20.315%
  手取り = 売却額 − 税額
  ```
- `capitalGainsThisYear` を追跡（→ P6の国保計算で使用）

**完了条件:**
- [ ] 現金・株式が分離して管理される
- [ ] NISA優先売却が機能する
- [ ] iDeCoが60歳未満で取り崩せない
- [ ] 課税口座売却時に20.315%課税が適用される

### Phase 6: FIRE後税金・社会保険

> P4D（売却益追跡）が前提

**変更内容:**

- **国民年金保険料**: FIRE後60歳未満は 16,980円/月 を支出に加算
- **国民健康保険料（動的計算）**:
  ```
  所得割 = max(0, 前年合計所得 − 430,000) × 11.0%
  固定   = 130,000円（2人世帯: 50,000×2人 + 30,000世帯）
  上限   = 1,060,000円/年
  前年合計所得 = セミFIRE労働収入 + capitalGainsLastYear
  ```
- **FIRE初年度スパイク**: 前年の給与所得が高いため、FIRE翌年の国保が急増する初年度のみ別計算
- `capitalGainsLastYear` を年次で繰り越して国保計算に使用

**完了条件:**
- [ ] FIRE後60歳未満に国民年金保険料が支出に加算される
- [ ] 国民健康保険料が前年所得ベースで動的に計算される
- [ ] FIRE初年度の国保スパイクが正しく計算される

**完了条件:**

## 注意事項

### 破壊的変更の一点集中

**Phase 1 は唯一の「破壊的変更」フェーズ**。`grossIncome` への移行をここで完全に完了させ、それ以降は後方互換を維持する設計とする。後回しにすると P2C・P3・P6 の実装時にすべて再修正が必要になる。

### テスト戦略

各Phase実装後、既存の66テストが全てパスすることを確認。Phase 1完了時は期待値の大幅更新が必要。新機能は必ず対応するテストを追加してから次のPhaseに進む。

### 段階リリース候補（P1完了後に先行リリース可能）

- **P2B** 住宅ローン — 単独で価値がある、実装コスト小
- **P2D** 児童手当 — 単独で価値がある、実装コスト小
- **P5** セミFIRE — 競合全社が対応済み、実装コスト小

### 後方互換の維持方針

- `Person.currentIncome` → P1で `grossIncome` に置換。`DEFAULT_CONFIG` 更新。
- `monthlyExpenses` → P2A後もオプションとして残す（子なし世帯はこちらを使用）。
- `Person.pensionAmount` → P3後も固定値指定が可能（指定がある場合は優先）。
