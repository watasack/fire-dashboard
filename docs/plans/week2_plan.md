## Day 1：購入状況の確認

### タスク：初週の数字を確認（15分）

| 確認項目 | 確認場所 |
|---------|---------|
| デモ利用数 | Streamlit Cloud ダッシュボード |
| note 記事 PV・購入数 | note クリエイタースタジオ |
| メール登録数 | Google フォーム 回答数 |

**判断基準：**
```
デモ利用あり・購入ゼロ → 記事の有料部分冒頭のコピー（何が入っているか）を改善
デモ利用自体が少ない  → X/note の発信量を増やす
メール登録ゼロ        → フォームの文言・設置位置を見直す
```

## Day 4：note 記事2本目 執筆

### タスク：SEO記事の執筆（2〜3時間）

**タイトル（案）：**
「共働きFIREシミュレーション｜育休・時短の影響を正確に計算する方法」

**狙うキーワード：** 「共働き FIRE シミュレーション」「育休 FIRE 計算」

**構成：**
1. 共働きFIREで見落とされがちなポイント（育休・時短・保育料）
2. 一般的なFIREシミュレーターの問題点
3. 育休を正確に反映するとどう変わるか（具体例）
4. デモで試してみてください（無料デモへの誘導）
5. 詳細計算はフル版（note 有料記事URL）

**文字数目標：** 2,000〜3,000字
**公開場所：** note（無料記事・SEO流入→有料記事誘導）

## Day 5：レビュー

### タスク：数字を確認して次の施策を決める（15分）

**判断フロー：**
```
購入 1件以上
  → 購入者にお礼メッセージ（note のメッセージ機能）
  → 「何が決め手でしたか？」を1問だけ聞く

購入 0件・デモ利用あり
  → 有料記事の冒頭コピーを修正（何が手に入るかをより具体的に）

デモ利用 < 20件/週
  → X 投稿頻度を週3回に増やす
```
