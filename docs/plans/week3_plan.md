## [転換率改善フロー] 購入0件・デモ利用ありの場合

デモを使ったのに買わなかった = 「欲しいが¥980の価値を感じなかった」

### 確認ポイント

```
1. note 有料部分の冒頭に何が書いてあるか（価値が伝わっているか？）
   → 改善例: 「アクセスコード + フル版URL + 3つの特典」を冒頭に明示

2. note 無料部分が「スクリーンショットなし・テキストのみ」になっていないか
   → docs/screenshots/ の画像が正しくアップロードされているか確認

3. デモの結果画面に「フル版で○○もできます」の訴求があるか
   → demo_app.py の育休ティーザーセクションが表示されているか確認
```

### タスク：note 記事の冒頭コピー改善（1時間）

有料部分の最初の3行を以下の形式に変更する：

```
【フル版Webツールへのアクセスはこちら】
URL: https://fire-dashboard-n4bdcefvcuu3ytl68fwtrw.streamlit.app/
アクセスコード: [購入者専用コード]

このコードで以下の機能が全て使えます：
✅ 育休・産休・時短を正確に反映したMCシミュレーション（1,000回）
✅ FIRE成功確率 / 最悪ケース資産推移グラフ
✅ 育休あり・なしの経済的インパクト比較
```

## アクセスコード管理フローの確立

week1 のブロッカーとして残っている「コード管理方法」を決定する。

### 推奨：シンプルな手動運用（Phase 1 では十分）

| ステップ | 内容 |
|---------|------|
| 1. 購入通知を受け取る | note からメール通知が来る |
| 2. 新しいコードを発行する | `ACCESS-{年月}-{連番}` の形式（例: `ACCESS-2603-001`） |
| 3. Streamlit Cloud のシークレットに追加 | share.streamlit.io → Settings → Secrets → `access_codes` リストに追記 |
| 4. note のメッセージ機能でコードを送付 | 件名「FIREシミュレーター アクセスコードのご案内」 |
| 5. 台帳に記録 | ローカルの Excel/スプレッドシートで管理（コード・日付・購入者メモ） |

**コード形式の例：**
```toml
# .streamlit/secrets.toml（Streamlit Cloud 上で設定）
access_codes = [
  "DEV-LOCAL-ONLY",
  "ACCESS-2603-001",
  "ACCESS-2603-002",
]
```

> **Phase 2（購入者が20件を超えたら）** 以降に自動化を検討する。
> 現段階では手動で十分。
